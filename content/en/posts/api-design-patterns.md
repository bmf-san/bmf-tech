[API Design Patterns](https://amzn.to/3AD3Vhr) is a book I read.

It starts with the principles of API design and introduces a wealth of API design patterns.

- Resource-oriented
  - For a single resource, determine the processing by combining standard methods (e.g., Create/Get/List/Delete/Update)
- RPC-oriented
  - Specify a method to call a specific procedure and determine the processing
- Considerations for improving API design
  - Executable
  - Expressive
  - Simple
  - Predictable

API design patterns provide many approaches that may be useful for challenges faced in API design and operation. It's a good reference when you're unsure about your design.
