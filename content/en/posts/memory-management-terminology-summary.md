# Overview
I was reading the memory management chapter of [Understanding Linux's Mechanisms](https://gihyo.jp/book/2018/978-4-7741-9607-7) and found some terms that I didn't fully understand, so I picked a few and summarized them.

# OOM
- Out of memory
- A state where the system has exhausted both physical and virtual memory and cannot allocate new memory.
- OOM Killer (Out of Memory Killer)
  - A mechanism in the Linux kernel that frees up memory by forcibly terminating processes during an OOM situation.

# Virtual Memory
- Memory that is virtualized to appear as the main memory to the OS and others.
- Allows disparate addresses on physical memory to be treated as sequential addresses.
- A function that considers a portion of the hard disk as addresses for virtual memory.
  - → Swap

# Page Table
- A data structure used in the OS's virtual memory to map virtual addresses to physical addresses.

# Page Fault
- An interrupt or exception generated by hardware when a program accesses a page in the virtual address space that is not mapped to physical memory.

# File Mapping
- Mapping a file into memory.
  - In C, this can be done using mmap.
    - This is similar to paging and swapping performed by the OS, but done within the process itself.

# Demand Paging
- A type of memory management method.
- Allocates physical memory to a page when the content of the corresponding address is needed.

# Copy-On-Write
- A mechanism that creates a copy upon writing.
- When generating a child process from a parent process, it temporarily shares the parent's memory space (with write protection specified) instead of copying all data in the parent's memory, reducing the processing time for process creation.
- When either process issues a write request to the shared memory space, a new memory space is allocated, and the data in memory is copied.

# Swap
- A feature of the OS that secures a storage area called a swap file or swap space on auxiliary storage like a hard disk to increase memory capacity.
- In the context of paging virtual memory, swapping or page swapping refers to exchanging pages between physical memory and auxiliary storage.
