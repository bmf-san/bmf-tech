# Overview
This is a note on the feature comparison when considering switching from New Relic to Datadog APM.

*Includes aspects beyond APM.*

# Comparison
|                        |New Relic                                                                                                                |Datadog                                                                                                            |Comparison                                                                                                                                                                            |
|------------------------------|-------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|Server Monitoring                |Metrics collection for cloud, containers, OS, middleware, network, OSS, etc. Metrics collection for CPU, memory, storage, network, processes, etc.                                            |Same as New Relic, so omitted                                                                                                  |No significant differences seem to exist.                                                                                                                                                                     |
|Application                      |Response time, throughput, error rate, transaction thread profiling, cross-application tracing, transaction analysis, service map, external service performance monitoring, transaction metrics and traces, distributed tracing, deployment analysis|Service map, service performance dashboard, deployment tracking, application logs, RUM, Synthetic integration, profiler (identifying code lines consuming CPU, memory, I/O), distributed tracing               |New Relic appears to be multifunctional, but Datadog seems capable of doing almost the same things.                                                                                                                                      |
|Database                        |Time consumed by database calls, slow queries, display of database and cache operations, response time and throughput of database calls, SQL query analysis (display of poorly performing queries and their stack traces)                      |Query throughput, performance, connection count, and other metrics can be obtained                                                                                   |Overall, there seems to be no significant difference. Datadog requires integration for MySQL. New Relic allows drill-down of logs through APM integration. Datadog likely can do similar things with RUM integration..                                                                      |
|RUM (Realtime User Monitoring)|Page load performance, browser performance, AJAX analysis, JavaScript error analysis, can integrate with New Relic Mobile                                                       |Metrics related to page views can be collected, RUM Explorer (access to all data collected from the application, useful for monitoring and error resolution), can integrate with logs, APM, and profiler, error tracking, iOS and Android supported via SDK|The frontend seems to be more robust in New Relic. Datadog appears to require additional JS tracer for JS-related monitoring. cf. https://docs.datadoghq.com/ja/tracing/setup_overview/setup/nodejs/?tab=%E3%82%B3%E3%83%B3%E3%83%86%E3%83%8A|
|Synthetic                     |Tests can be conducted by location (worldwide), can specify daily, every minute, etc., availability monitoring, scripts, browser interactions (mysterious), API tests, possible integration with APM? (probably), CI/CD integration possible                        |Browser tests, private locations (monitoring internal applications, monitoring private URLs that cannot be accessed from the internet), CI/CD integration, can integrate with APM                                     |No significant differences seem to exist.                                                                                                                                                                    |
|Reporting                          |Custom dashboards, availability, capacity analysis, SLA, deployment tracking, usage analysis per host, scalability                                                                           |Custom dashboards, predictive monitoring, anomaly detection monitoring, deployment tracking, SLO                                                                            |Datadog seems to focus on monitoring, while New Relic appears to emphasize analysis.                                                                                                                                    |
|Integration Features                          |Can integrate with various cloud services and middleware                                                                                                |Same as New Relic, so omitted                                                                                                  |Both have scalability.                                                                                                                                                                   |
|APM Installation Method                       |Install agents for each language                                                                                                       |Install Datadog Agent. Enable APM in Datadog Agent settings, configure tracers for each language                                                    |New Relic APM has information about agent CPU load, but Datadog APM is unclear. Needs verification.
