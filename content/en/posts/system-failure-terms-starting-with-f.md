# Overview
This is a note as a glossary to avoid confusion.

# Terms
## Fail-Safe
A mechanism that transitions to a safe state when a failure occurs.

## Failover
A system that automatically transfers functionality to a backup system and continues processing when a failure occurs in the active system.

## Failback
The process of transferring operations back from the backup system to the original system, returning to the original state. The reverse of failover.

## Fail-Safe
Continuing operation by removing the failure point and narrowing the impact when a failure occurs.

## Fault-Tolerant
Maintaining normal operational status by switching to a backup system when a failure occurs.

# Fault Avoidance
The concept of eliminating causes that lead to system failures or faults.

## Fallback
Maintaining system availability by limiting functions or performance or switching to another system when a failure occurs. Degraded operation.

## Foolproof
Designing the system so that users cannot perform incorrect operations or dangerous usage. Also refers to such mechanisms or structures.

## Fault Masking
Creating a mechanism so that the impact does not spread even if a failure occurs.
