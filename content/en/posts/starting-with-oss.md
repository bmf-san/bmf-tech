I have been developing a CMS while publishing the source code on GitHub as OSS for the past few months. I will ramble on about the purpose of starting this.

# What is OSS?
Let's refer to Wikipedia for the definition of OSS.

> Open-source software (OSS) is software with source code that is made available for use, modification, and distribution under a license that grants these rights to the copyright holder for any purpose. Open-source software is developed collaboratively. Open-source software is often compared to consumer-generated media (technically defined) and the open content movement (legally defined). [Wikipedia](https://ja.wikipedia.org/wiki/%E3%82%AA%E3%83%BC%E3%83%97%E3%83%B3%E3%82%BD%E3%83%BC%E3%82%B9%E3%82%BD%E3%83%95%E3%83%88%E2%80%8C%E3%82%A6%E3%82%A7%E3%82%A2) 

In summary, it is software that has the following characteristics:

- Source code is publicly available
- Collaborative development
- Free license

I should mention that the product I am currently developing does not fully meet the requirement of "collaborative development," so it might be difficult to strictly call it OSS, but please overlook the details.

# What am I developing?
I am developing a CMS that allows writing articles in Markdown. The backend is developed with Laravel, and the frontend is developed with React.

![screencapture-admin-rubel-dashboard-new-post-1500641283831.png](https://qiita-image-store.s3.amazonaws.com/0/124495/5b01469f-b8d2-b023-f975-0a9d649d8788.png)

### Motivation for Development
#### "I wanted to have my own product"
- I want to develop something I would like to use in my own way.
- I want someone to use the product I developed.

#### "I wanted to improve my technical skills"
- I want to have opportunities to try new technologies.
- A place to practice what I have learned in work and private life.

I started with the mindset of "I create because I want to create, it's fun to create, and if the result leads to skill improvement, that would be great."

### Why a CMS?
- I wanted to stop using WordPress.
    - There are many features I don't use.
    - I want to write articles simply like on Qiita, and I want Markdown writing to be the default.
    - SPA.
    - If I'm going to write a blog, I want to use my own CMS.
- I can challenge myself with various feature implementations.
 - Recommendation features using machine learning.
 - Integrating with Amazon API to introduce reference books and affiliate features.
 - Bitcoin tipping feature.
 - Automatic posting to Twitter.
 - Scheduled posting feature.
 - Markdown import/export feature.
 etc...

There are many opportunities to try various technologies and ideas, making it enjoyable to develop.

# Why OSS?
- I want various people to review it.
- I want various people to use it.
- Portfolio.
    - The source code is open, so anyone can view it.

# What can be gained?
- By continuing development and operation in the long term, I can experience my own growth.
- Opportunities for code reviews from various people.
- The pressure of being seen by someone stimulates my desire to learn, leading to increased motivation.
- A mysterious confidence.
- Fulfillment of the desire for recognition (such as stars).

# Poetic Insight
I thought there is no need to feel guilty about starting OSS with a self-centered mindset, as solving my own problems could potentially lead to solving others' problems and contributing to them.

Since there weren't many development examples of Laravel and React on GitHub, I hope to receive various feedback.
