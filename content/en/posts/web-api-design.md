[Web API Design](https://amzn.to/3y7dqEG) is a book that provides perspectives on API design.

It discusses not only the interfaces of API design but also the entry points that come before it.

- It's better to design API focusing on what users can do rather than the mechanisms, as focusing on the mechanisms can lead to complexity.
- There was an explanation of a framework called Goal Canvas as an approach to clarify the goals of the API, which I thought was a good method.
- Patterns for API versioning include path, domain, query parameters, custom headers, content negotiation, and consumer settings (holding settings for each consumer in the database).
