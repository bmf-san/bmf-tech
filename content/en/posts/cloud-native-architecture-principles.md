[Cloud Native Architecture: Principles of Next-Generation Design for Availability and Cost-Effectiveness](https://amzn.to/446hhyM) is a book that allows you to understand the fundamental concepts of cloud native.

This book explains cloud native architecture based on the Cloud Native Maturity Model (CNMM).

The Cloud Native Maturity Model is a metric for evaluating cloud native maturity by following three design principles: "Cloud Native Services," "Application-Centric Design," and "Foundations of Automation."

The basic meaning of cloud native is to design by leveraging cloud computing services, which is only part of the requirements. In other words, it is not whether something is cloud native that is important, but rather whether the architecture is mature as a cloud native design.

This book has been out for some time, and many parts of it have become commonplace today. It is likely written against the backdrop of the transition from on-premises to cloud and the rise of cloud computing.

I have the impression that it is a good read to consider the question, "What is cloud native?"
