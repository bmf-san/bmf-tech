I created a repository to organize and preserve original specifications and frameworks I came up with.

**Repository**: [spec-by-bmf-san](https://github.com/bmf-san/spec-by-bmf-san)

# Background

During development and process improvement, I often feel that existing frameworks or methods could be improved. I wanted a dedicated place to document and preserve such ideas and improvement plans, so I decided to set up a repository.

# Currently Published Specifications

## bmf Framework

This is a new framework designed to address issues with the classic KPT (Keep / Problem / Try) retrospective method.

- **Build**: Visualize achievements first to promote positive retrospectives.
- **Miss**: Share problems without censorship, with the premise of selecting focus areas later.
- **Focus**: Replace "Try" with "Focus" to prioritize quality over quantity.

For more details, refer to the [bmf Framework Specification](https://github.com/bmf-san/spec-by-bmf-san/blob/main/specs/bmf-framework-ja.md).

## Story Point Estimation Method Using the Cynefin Framework

This method combines the Cynefin Framework's five domains with story point estimation.

By categorizing tasks into "Simple," "Complicated," "Complex," "Chaotic," and "Disorder," and applying estimation methods and point criteria suitable for each, it enables more accurate estimations.

For more details, refer to the [Cynefin Story Points Method Specification](https://github.com/bmf-san/spec-by-bmf-san/blob/main/specs/cynefin-story-points-ja.md).
